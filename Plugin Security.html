üìú Guide: Configuring plugin_security.json

üõ°Ô∏è What is this file?

This file is the Security Policy for your server.

When a plugin wants to do something‚Äîlike access the Database, read files, or send network requests‚Äîit acts like a guest at a hotel. This file acts as the Front Desk Manager. It tells the system:

"Which guests (Plugins) are allowed into which rooms (Services)?"

By editing this file, you can decide exactly what plugins are allowed to do on your server without writing a single line of code.

üìç Where is it?

The file is located in the Root Directory of your server application (the same folder as your .exe or appsettings.json).

File Name: plugin_security.json

Note: If this file is missing, the server will automatically create a default one when it starts.

‚öôÔ∏è How It Works (The "Master Switch")

At the top of the file, you will see a setting called "DefaultMode". This is the most important setting.

Option A: "Deny" (Recommended for Production)

json
"DefaultMode": "Deny"

Behavior: The server acts like a locked house. Nothing is allowed unless you specifically add a rule in the "Policies" list below.

  • Pros: Maximum security. Even if a malicious plugin is installed, it can't do anything until you approve it.
  • Cons: You must manually enable every service a plugin needs.

Option B: "Allow" (Good for Development)

json
"DefaultMode": "Allow"

Behavior: The server acts like an open park. Everything is allowed unless you specifically block it.

  • Pros: Easy to develop plugins. They just work.
  • Cons: Less secure. A bad plugin could delete files or crash the database.

üîë The Permission Numbers (The Keys)

When creating a rule, you must assign a "Required Permission" number. This number represents a Flag. You can add these numbers together to combine permissions.

Permission Name
Value
Description
DatabaseAccess1Allows access to the Database (e.g., FeatherDatabase, UserService).
NetworkAccess2Allows internet requests (e.g., HttpClient).
FileSystemAccess4Allows reading/writing files on the server.
EventBusAccess8Allows listening to or publishing server events.

How to Combine: If you want a plugin to have Database access AND Network access, you add the numbers: 1 + 2 = 3.

üìù Example Configurations

Example 1: The "Iron Fortress" (Secure)

You want to run a public server. You only want plugins to see the Database, but never touch the file system or network.

json
{
"DefaultMode": "Deny",
"Policies": [
{
"ServiceType": "SharpPress.Services.FeatherDatabase",
"RequiredPermission": 1
},
{
"ServiceType": "SharpPress.Services.UserService",
"RequiredPermission": 1
}
]
}

Result: Plugins can access the database, but any attempt to use HttpClient (Network) or File.ReadAllText (FileSystem) will be Blocked immediately.

Example 2: The "Full Access" Plugin

You have a trusted plugin called "MySuperPlugin" that needs to manage the database, download updates (Network), and write logs (FileSystem).

json
{
"DefaultMode": "Deny",
"Policies": [
{
"ServiceType": "SharpPress.Services.FeatherDatabase",
"RequiredPermission": 5
}
]
}

Note: 5 means Database (1) + FileSystem (4). Result: This plugin can read/write DB and Files. It cannot access the Network (because 5 does not include the number 2).

Example 3: The "Developer Mode" (Easy)

You are developing plugins and don't want to be bothered with permissions.

json
{
"DefaultMode": "Allow",
"Policies": []
}

Result: Plugins can do whatever they want.

üö® Troubleshooting & Common Errors

Error: "UnauthorizedAccessException"

What happened? A plugin tried to use a service, but it didn't have the correct permission.

How to fix:

  1. Check the error log to see which ServiceType caused the crash (e.g., SharpPress.Services.UserService).
  2. Open plugin_security.json.
  3. Find the policy for that ServiceType.
  4. Ensure the RequiredPermission number matches what you assigned to the plugin.
  5. Restart the server.

Error: "Could not find type..."

What happened? You typed the name of the service incorrectly.

How to fix:

  1. You must use the Full Name (Namespace + Class Name).
  2. Example: Don't write "UserService". Write "SharpPress.Services.UserService".
  3. If you don't know the full name, open your server project in Visual Studio and right-click the class -> "Go to Definition".

üîÑ Applying Changes

Important: Changes to this file do not update instantly while the server is running.

  1. Edit the file.
  2. Stop the server.
  3. Start the server.
  4. The server will load the new security policy on startup.

üìö Quick Reference Cheat Sheet

To allow ONLY Database: "RequiredPermission": 1

To allow ONLY Network: "RequiredPermission": 2

To allow ONLY File System: "RequiredPermission": 4

To allow ONLY Events: "RequiredPermission": 8

To allow EVERYTHING (God Mode): "RequiredPermission": 15 (1+2+4+8)

To allow NOTHING (Locked): "DefaultMode": "Deny" (and no policies listed)
